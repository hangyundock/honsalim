# CATEGORY_PAGE.md — 카테고리 페이지 표준 템플릿 (정형화) — 세션 #14 확정

> 혼살림은 **카테고리 우선 제품 비교·정보 사이트**(노써치형)로 전환 확정 (DECISIONS O2).
> 이 문서 = 모든 품목 카테고리 페이지가 따르는 **표준 규격**. 새 카테고리는 검색어·가격밴드만 지정하면 이 틀로 생성.
> 시각 프로토타입(실데이터·하드코딩) = `scripts/category_page_prototype.py` (실행 시 흰 바탕 시안 HTML 생성).
> ⚠️ **현재는 목업 단계** — "검색어만 주면 자동 생성"은 §6 본구현 완료 후 가능.

## 0. 정체성 (DECISIONS O2)
- 우리 유형 = **용도·라이프스타일 맞춤 큐레이션 + 구매가이드 + 기능/가격대 비교**.
- 노써치와 다른 점(정직): 단일 판매처(알리)·직접 테스트 불가 → **멀티벤더 최저가·랩테스트형 아님.**
- 차별점 = "어떤 사람에게 어떤 제품이 맞나" 큐레이션 + 전문 정보 번역.
- 세팅("홈오피스 50만원")은 **보조** = 카테고리 가이드를 잇는 큐레이션 경로(침대→이불 식 크로스링크).

## 1. 디자인 (DECISIONS O3)
- **노써치식 담백한 흰 바탕** (우드톤/Claude풍 폐기).
- 폰트 **NanumSquare Neo**(네이버 제작·무료, 웹폰트). [추정 — 네이버 채용페이지 폰트, 직접 inspect 미확정]
- 텍스트색 찐회색(meta `#5a626b`/sub `#3f464e`), 너무 검정 X.
- 배너·개념 일러스트는 **디자인 AI로 생성**(클로드코드가 못 뽑음). 제품 사진은 **알리 실사진 핫링크**.
- **AI가 썼다는 표현/인식 절대 금지** — 저자 = 운영자 "혼살다". AI 표기는 *개념 이미지*에만.

## 2. 페이지 구조 (위→아래, 12컴포넌트)
1. **공정위 고지 배너** — 최상단 고정(2024.12.1 개정: 제목/첫부분 의무). 위치·형식 변경 불가.
2. **타입 선택기** — 카테고리 내 타입 썸네일(예: 의자 = 안장형·무릎꿇이·게이밍 + 실속/고급) + 끝에 **"+더 많은 제품 보기"** 카드.
3. **제목 + 저자(혼살다) + 업데이트 날짜** (+ 공유·댓글).
4. **도입 prose**(문제 제기/공감) + **신뢰 박스 2개**:
   - 🤝 혼살림의 약속(제조사 광고비·협찬 안 받음 / 제휴 수수료는 받지만 추천 선정과 무관 — *상단 고지와 모순 없게 연결*)
   - 📋 이렇게 골랐어요(판매자 표기 스펙·옵션·가격을 1인가구 관점 비교. **"후기" 주장 금지** — 알리 API에 후기 없음. 가격·할인 수집시점 명시)
5. **섹션 헤더 스타일** = 왼쪽 강조선 + eyebrow + 제목. (노써치 검정박스·"꼭 알아야 할~기본정보" 카피 금지 — 독창화)
6. **기본 정보(전문 8요소)** — §3. 타입 차이 표 + 체크포인트(왜 중요) + 흔한 실수. 개념 이미지(디자인 AI) 자리.
7. **추천 2분류 비교 카드** — **💰 실속형 / ⭐ 고급형** 2단 컬럼(좌/우). 카드: 이미지·이름·가격·**정가→판매가+할인율**·장점(＋)/단점(－)·추천대상·[더 알아보기]·[최저가/구매]. (고급형 위로 3티어 프리미엄은 1인가구엔 과해 제외)
8. **한눈에 비교표** — 실속+고급 전체 스펙 매트릭스(O/△/X), 가로 스크롤. 티어 배지.
9. **더 많은 제품 보기** — "전체 제품" 탭(필터·정렬 랭킹 리스트)으로 연결. 타입선택기 끝·본문 중간·하단 3곳 노출. (§6 본구현 — 다음 세션)
10. **FAQ** (롱테일 검색).
11. **고지·사업자 정보** 푸터.
12. 서브탭: 통합 / 추천·비교 / 구매가이드 / 전체 제품.

## 3. 콘텐츠 8요소 (혼살다 명의·전문가급 정보) (DECISIONS O5)
1. 스펙 해독(용어·숫자 → 의미) 2. 기준+"왜"(메커니즘) 3. 트레이드오프 4. 용도→타입 매핑
5. 흔한 실수·함정 6. 정직한 단점 7. 객관 기준·인증 인용(KC·에너지효율·헤파 등) 8. 1인가구·예산·공간 렌즈(=commodity 회피).
- **가드레일**: 추측 금지·등급 표기 / 의료·효능 단정 금지 / 과장·최상급 금지 / **1인칭("써봤다") 금지** / 일반지식 베껴쓰기 금지 / **경쟁사 문장 카피 금지(구조만 참고, 표현 100% 원본)**.

## 4. 신뢰 신호 = 정가→판매가+할인율 (DECISIONS O6)
- 알리 API(product.query=productdetail.get) 제공: 이름·**판매가/정가/할인율**·이미지(메인+추가 갤러리)·영상(있을 때)·2단계 카테고리·판매처·판매량(불안정, 0 잦음).
- **미제공(=쓰면 안 됨)**: 평점·**긍정평가율**·구조화 상세스펙·후기 텍스트. → 가짜 평점 금지(§진실성).
- 할인 0%면 신호 없이 가격만(무릎꿇이 사례).

## 5. 가격 드리프트 규칙 (DECISIONS O7) ★중요
- 알리 가격 변동 큼 — **게이밍 의자 105,200 → 239,264원(2배) 실측**. 저장값 stale.
- 필수: ①빌드/표시 시 가격 재조회 ②"○○ 기준" 시각 표기 ③진실성 게이트로 변동 거름 ④**가격대 이탈 시 자동 재선정**(티어 밖으로 나가면 다른 제품 교체).
- 사유: 틀린 가격 = 신뢰 붕괴 + 공정위 가격정확성 위반.

## 6. ★ 본구현 남은 것 (다음 세션 핵심 — "자동화")
지금은 `scripts/category_page_prototype.py`에 **손으로 넣은 데이터**로 만든 목업. 새 카테고리를 검색어만 주고 자동 생성하려면:
1. **콘텐츠 파이프라인** — 혼살다 명의로 AI가 8요소 가이드·제품별 장단점·추천대상 자동 작성 + 진실성 게이트. (enrich 프롬프트 개편)
2. **카테고리별 2티어 수집** — 구체 검색어 + 실속/고급 가격 밴드. `search_keywords.yml`에 카테고리×티어 밴드 정형화. (교훈: "office chair"+저밴드=액세서리, "ergonomic mesh office chair"+45~100k=진짜 의자)
3. **수집기 필드 추가** — `target_original_price`·`discount` (+ DB 컬럼) → 신뢰 신호. 가격 빌드시 재조회.
4. **렌더러 이식** — 이 프로토타입을 Jinja 템플릿 + `builder/renderer.py`로. (article.html과 별개 category 템플릿)
5. **"전체 제품" 더보기 페이지** — 필터·정렬 랭킹 리스트.
