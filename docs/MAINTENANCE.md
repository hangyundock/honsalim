## 2. 의존성 업데이트 정책

### 2-1. Python 패키지 분류

| 분류 | 패키지 | 업데이트 정책 |
|------|--------|-------------|
| 핵심 | anthropic, jinja2, requests, python-dotenv | 마이너 업데이트만 자동, 메이저는 수동 검토 |
| 보조 | markdown, pyyaml, pillow | 분기 1회 일괄 |
| 개발 도구 | black, ruff, mypy, pytest | 자동 (lint·테스트 통과 시) |
| 신중 | (없음) | — |

### 2-2. 업데이트 주기

| 주기 | 작업 |
|------|------|
| 매 push | Dependabot 자동 PR (공개 저장소) |
| 주 1회 | Claude가 PR 검토·테스트 통과 시 머지 안 제시 |
| 월 1회 | 사용자가 수동 검토 후 일괄 머지 (사용자 승인) |
| 분기 1회 | 메이저 버전 검토 (breaking change) |

### 2-3. 머지 게이트

- pytest 통과 (validator 회귀 100%)
- mypy 통과
- 로컬 빌드 정상 (build/ 생성 + 크기 동일성)
- 사용자 명시 승인

### 2-4. 핀 정책

- `pyproject.toml`은 minimum version 기재 (`>=`)
- `requirements.lock` 또는 `uv.lock`은 정확한 버전 핀
- 로컬 D 드라이브와 Actions 환경 일치 (ARCH §12-4)

### 2-5. 다운그레이드 절차

- 새 버전이 회귀 발생 시
- lockfile 이전 버전으로 revert
- pyproject.toml의 max version 제약 추가 (`<n.m.p`)

## 6. 사이트 마이그레이션 시나리오

### 6-1. Cloudflare Pages → 타사 이동 (희귀)

**가능성**: Cloudflare 측 정책 변경·가격 인상·서비스 다운 [관찰] (극히 드문).

**대안 후보**:
- GitHub Pages (정적, 무료, 단 한국 응답 속도 ↓)
- Vercel (정적/SSR, 한국 PoP 일부)
- Netlify (정적, 한국 PoP 일부)
- 자체 VPS (Cafe24·KT·NHN 등)

**판단 기준**:
- LCP ≤ 2.0초 (B6 [확정]) 유지 가능
- 무료 또는 월 10,000원 이하
- 자동 배포 (GitHub Actions 연동)

### 6-2. 마이그레이션 절차 (가정적) [추정]

| 단계 | 작업 | 기간 |
|------|------|------|
| 1 | 후보 1개 선택 + 트라이얼 계정 | 1일 |
| 2 | 빌드 산출물 정상 배포 검증 | 1일 |
| 3 | 도메인 DNS 일부 (개발 서브도메인) 전환 | 1일 |
| 4 | 1주일 운영 비교 (LCP·INP·CLS) | 1주 |
| 5 | 본 도메인 DNS 전환 (TTL 사전 단축) | 1일 |
| 6 | 7일 모니터링·rollback 준비 | 1주 |
| 7 | Cloudflare 자산 회수 (R2·D1) | 1주 |
| 8 | DECISIONS B3 폐기 + 새 호스팅 [확정] 등록 | 1일 |

### 6-3. 마이그레이션 회피 우선

- 본 시점: Cloudflare 그대로 (가격·기능·한국 PoP 모두 최선 [관찰])
- 마이그레이션은 절체절명 상황만 (정책 위반·가격 폭증)

### 6-4. R2 마이그레이션 (장기 가능)

- 자체 VPS + nginx + 별도 CDN
- 또는 AWS S3 + CloudFront (영어 확장 시)
- 비용 vs 운영 부담 검토

## 10. Cloudflare Pages 보조 배포 (B-3 미결정 사항)

### 10-1. 1차 결정 (ARCH 부록 B-3, OPS §11-3)

- 본 시점: **미적용**
- Cloudflare Pages SLA·자동 복구 신뢰 [관찰]
- 보조 배포는 운영 복잡도 ↑

### 10-2. 재검토 시점

- 6개월 운영 후 다운타임 누적 ≥ 1시간
- 또는 사용자 트래픽 100+/일 도달 시 (다운의 영향 ↑)

### 10-3. 보조 배포 후보

| 옵션 | 동기 빈도 | 비용 |
|------|----------|------|
| GitHub Pages | 매 push (Actions 동시) | 무료 |
| Cloudflare R2 + 자체 도메인 | 매 빌드 | R2 비용 |
| Vercel | 매 push | 무료 한도 |

GitHub Pages가 가장 간단 (이미 GitHub 사용 중) [추정].

### 10-4. 활성 시 절차

| 단계 | 작업 |
|------|------|
| 1 | GitHub Pages 활성 (저장소 Settings) |
| 2 | Actions 워크플로 추가 (build/ → gh-pages 브랜치) |
| 3 | 대체 도메인 (예: backup.honsalim.com) 발급 |
| 4 | DNS A/AAAA 보조 (낮은 priority) |
| 5 | Cloudflare Pages 다운 감지 시 DNS swap 절차 문서화 |
