## 2. 브랜드 컨셉

### 2-1. 한 줄 정의

> **혼살림 = "혼자 살림을 따뜻하게 시작하는 사람을 위한 깔끔한 안내자."**

### 2-2. 컨셉 3축 (DECISIONS A6 [확정])

| 축 | 방향 | 시각적 표현 |
|----|------|------------|
| **미니멀** | 군더더기 0·정보 위계 명확 | 충분한 여백·1~2개 강조 색·서체 위계 단순화 |
| **따뜻함** | 차갑지 않은 1인 가구·정착감 | 우드 톤 액센트·부드러운 그림자·둥근 모서리 |
| **신뢰** | 어필리에이트 추천 사이트의 정직함 | 첫머리 disclosure·1인칭 자제·등급 표시 |

### 2-3. 벤치마크 (DECISIONS A6·G1 [확정] + L4 [확정 세션 #6 차용 범위 명문화])

## 4. 한글 타이포그래피 가이드

### 4-1. Pretendard 적용 [확정]

```
<link rel="preload" as="font" type="font/woff2"
      href="/static/fonts/Pretendard-Variable.woff2" crossorigin>

font-family: "Pretendard Variable", Pretendard, -apple-system,
             BlinkMacSystemFont, system-ui, sans-serif;
font-display: swap;  /* LCP 보호 (B6 [확정]) */
```

이유 [추정]:
- 한국어 가독성 1위급 [관찰]
- Variable Font → weight 슬라이더 부담 0
- 무료 SIL OFL 라이선스
- woff2 단일 파일 ~110KB (preload 가능)

### 4-2. 한글 본문 룰

| 룰 | 값 | 이유 |
|----|----|------|
| `letter-spacing` | -0.01em | 한글 답답함 완화 |
| `word-break` | keep-all | 단어 단위 줄바꿈 (한글) |
| `overflow-wrap` | anywhere | 긴 URL 깨짐 방지 |
| `line-height` (본문) | 1.75 | 한글 가독성 [관찰] |
| `max-width` (본문) | 65~72ch | 한 줄 28~32자 (한글 권장) |

### 4-3. 숫자·가격 표기

- 가격: `1,200,000원` 쉼표 + "원"
- 천 단위 구분: 한국어 표준 (1,200원 / 12,000원 / 120,000원)
- 단위 강조: 가격 숫자 `font-semibold`, "원" `font-regular`
- 영문 수치 (예: 27인치)는 그대로

## 5. 컴포넌트 라이브러리

페이지를 만드는 18개 컴포넌트. 모두 모바일 우선 설계.

### 5-1. 글로벌 (전 페이지)

#### `header`
- 좌: 브랜드 마크 (혼살림 + wood-700 마크) — 홈 링크
- 중: nav (홈·페르소나·시나리오·About) — 데스크톱만
- 우: 검색 아이콘 (Phase 4 이후)
- 모바일: 햄버거 메뉴 (Phase 4 이후)
- 높이: 64px / sticky

#### `footer`
- 4컬럼 (페르소나·시나리오·정보·법무)
- disclosure 풀 문구 (E1)
- 사업자 정보 (E3, 등록 후)
- 카피라이트 + 운영자 메모

### 5-2. 콘텐츠 컴포넌트

#### `disclosure-banner` (E1 핵심)
- 글 첫머리 sticky 카드
- wood-100 배경 + wood-700 텍스트
- 문구: "이 글에는 쿠팡 파트너스 활동의 일환으로 일정 수수료를 제공받습니다. (구매자 추가 부담 없음)"
- 닫기 버튼 없음 (의무 표시)

#### `hero-article`
- 페르소나 태그 + 시즌 태그 + 예산 태그 (3개 칩)
- 페이지 제목 (text-4xl, font-bold)
- 요약 (text-lg, ink-700)
- 사용자 직접 사진 1장 (16:9, rounded-lg)
- 메타 (작성일·수정일·읽기 시간·조회수)

#### `product-card`
- 좌: 이미지 (1:1 정사각, R2 또는 쿠팡 공식 위젯)
- 우상: 상품명 (text-lg, font-semibold) + 가격 + 가격 변동 안내
- 우중: 추천 메모 (200~300자, ink-700)
- 우하: 1인칭 한국어 짧은 코멘트 (옵션, 직접 사진 시)
- 우최하: CTA "쿠팡 보러 가기 →" (wood-500 → wood-700 hover)
- rel="nofollow sponsored" 자동
- 클릭 시 `/go/<slug>` 게이트웨이

#### `budget-table`
- 6~12개 상품의 예산 분배 표
- 좌: 카테고리 / 중: 추천 / 우: 가격 / 우우: 누계
- 마지막 행: 합계·예산 차액
- 모바일: 카테고리 + 가격만 (간소화)

#### `faq-section`
- 3~5개 FAQ
- 토글 (details/summary 네이티브)
- 검색 친화 (Schema.org FAQPage는 Phase 5 이후 검토)

#### `related-scenarios`
- 같은 페르소나 또는 같은 시즌 시나리오 2~3편
- 카드 그리드 (모바일 1열 / 데스크톱 3열)

### 5-3. 목록·네비게이션

#### `scenario-card`
- hero 이미지 (16:9, lazyload)
- 페르소나 태그 + 예산 태그
- 제목 + 1줄 요약
- 메타 (작성일·조회수)

#### `persona-hub`
- 페르소나 hero (사진 + 1줄 설명)
- 해당 페르소나 시나리오 카드 리스트
- 사이드: 페르소나 검색어·구매 채널·시즌 캘린더

#### `breadcrumb`
- Home > 페르소나 > 시나리오 (BreadcrumbList Schema [확정])
- 모바일 첫 항목 줄임 (... > 페르소나 > 시나리오)

#### `tag-chip`
- 둥근 모서리 + wood-100 배경
- 태그 텍스트 + 옵션 아이콘
- 클릭 시 태그 필터링 (Phase 4)

### 5-4. UI 기본

#### `button-primary`
- wood-500 배경 + paper 텍스트
- hover wood-700
- padding 12px 24px, radius-md
- font-medium

#### `button-ghost`
- 투명 배경 + wood-700 텍스트
- hover wood-100 배경

#### `input-search`
- Phase 4 이후

#### `pagination`
- 글 목록 페이지 (10편/page)
- 1 < ... 4 5 6 ... 12 >

### 5-5. 진실성 표시 컴포넌트

#### `truth-badge`
- 글 메타에 [확정] [관찰] [추정] 등급 표시 (옵션)
- 작은 칩 형태

#### `personal-photo-badge`
- 직접 촬영 사진에 작은 "직접 촬영" 마크
- 신뢰 강화

#### `price-update-note`
- "가격은 YYYY-MM-DD 기준입니다" 자동 삽입
- 모든 product-card 하단

## 7. 반응형 브레이크포인트

### 7-1. 모바일 우선 (스타팅 포인트)

| 브레이크포인트 | min-width | 용도 |
|---------------|-----------|------|
| `xs` (default) | 0 | 모바일 (~430px 폰) |
| `sm` | 640px | 큰 모바일·작은 태블릿 |
| `md` | 768px | 태블릿 세로 |
| `lg` | 1024px | 태블릿 가로·작은 데스크톱 |
| `xl` | 1280px | 데스크톱 |
| `2xl` | 1536px | 큰 데스크톱 |

### 7-2. 컨테이너 max-width

| 브레이크포인트 | max-width |
|---------------|-----------|
| `xs` | 100% (padding 16px) |
| `sm` | 100% (padding 24px) |
| `md` | 720px |
| `lg` | 960px |
| `xl` | 1120px (글 본문 maxw 720px 별도) |

### 7-3. 핵심 디자인 가정

- 1차 트래픽 모바일 60%+ [관찰] → 모바일 우선
- 글 본문 폭은 데스크톱에서도 720px 이하 (가독성)
- 사이드바·필터는 lg 이상에서만 표시

## 9. CWV 디자인 측면 (B6 [확정] 목표 달성용)

### 9-1. LCP ≤ 2.0초

| 룰 | 적용 |
|----|------|
| Hero 이미지 preload | `<link rel="preload" as="image">` |
| Pretendard preload | `<link rel="preload" as="font">` |
| Critical CSS 인라인 | 첫 화면 분량만 inline, 나머지 defer |
| 이미지 fetchpriority="high" | hero 이미지 한정 |

### 9-2. INP ≤ 150ms

| 룰 | 적용 |
|----|------|
| JS는 defer | 모든 script defer 또는 type=module |
| 이벤트 핸들러 가볍게 | 무거운 작업 requestIdleCallback |
| 라이브러리 의존 최소 | jQuery·React 미사용 (Phase 4까지) |

### 9-3. CLS ≤ 0.05

| 룰 | 적용 |
|----|------|
| 모든 img width·height 속성 | images.width_px/height_px 강제 (DB §9) |
| 폰트 font-display: swap | FOIT 회피 |
| 광고·임베드 영역 reserve | 쿠팡 위젯 영역 placeholder 박스 |
| 동적 컨텐츠 위에서 삽입 금지 | 무한 스크롤 미사용 |
